Introduction

Gridicarus (Grid Icarus) is a CSS framework built around CSS Grids. Its main purpose is to build and prototype grid-based websites quickly and easily. And while it's still in development, it allows grid-based websites (like this one) to be created almost instantaneously without any setup. Take a look at the source code for this page to see how Gridicarus is being used!

Getting Started

To get started with Gridicarus, use either npm:

npm install gridicarus

Or the CDN link:

<link rel="stylesheet" href="https://unpkg.com/gridicarus@0.1.2/dist/gridicarus.min.css">

Grid-Parent

Defining a grid.

To create a new grid, simply use the .grid-parent class:

<div class="grid-parent">
	<div class="grid-child #1 box-blue">
		This is some content!
	</div>
</div>
This is some content!

This will create a 1:1 grid that densifies its children. A parent grid can structure its children using the following classes:

  • .h-fill(1-9) - Creates (1-9) empty horizontal cells.
  • .v-fill(1-9) - Creates (1-9) empty vertical cells.
  • .h-gap-size(1-9) - Adds a horizontal gap (rem) for each cell.
  • .v-gap-size(1-9) - Adds a vertical gap (rem) for each cell.

We can use these classes to control the flow of our grid layout or give the cells some spacing. Here's an example:

<div class="grid-parent h-fill2 h-gap-size9">

  <div class="grid-child #1 v-span1 box-lime">
  	Area Left
  </div>
  <div class="grid-child #2 v-span1 box-blue">
  	Area Right
  </div>

</div>
Area Left
Area Right

We use .h-fill2 to specify that we'd like two horizontally stacked cells, and .h-gap-size9 to specify that we'd like the smallest (9) amount of spacing between cells.

Gridicarus uses "h-styled" sizing. This means that the largest number is the smallest size, and the smallest number is the largest size:

  • (v/h)-gap-size1 = 3rem
  • (v/h)-gap-size2 = 2.5rem
  • (v/h)-gap-size3 = 2rem
  • (v/h)-gap-size4 = 1.5rem
  • (v/h)-gap-size5 = 1.25rem
  • (v/h)-gap-size6 = 1rem
  • (v/h)-gap-size7 = 0.75rem
  • (v/h)-gap-size8 = 0.50rem
  • (v/h)-gap-size9 = 0.25rem

Grid-Child

Customizing the grid layout.

Once the grid is setup, we can begin to develop it further using the .grid-child class. If we'd like to control the order of how children are placed, every child in the grid must have the .grid-child class followed by the #1-10 class. You may also use .grid-child(1-9) instead. Here's an example of this website's layout using the options above.

To make children larger than others, use the .h-span(1-10) or .v-span(1-10) classes. Here's an example: 

<div class="grid-parent">

  <div class="grid-child #1 h-span2 box-lime">
    Header
  </div>
  <div class="grid-child #2 h-span2 box-blue"> 
    Body
  </div>
  <div class="grid-child4 box-grey">
    Footer Left
  </div>
  <div class="grid-child4 box-silver">
    Footer Right
  </div>

</div>
Header
Body
Footer Left
Footer Right

Notice that .h-span2 is used to dictate two children spanning 2/4 cells. We created four cells by having four children of the parent grid. If the number of children is even, .h-span(1-10) only needs to be added to the children we'd like to resize. If not, you must specify the size for each child. Gridicarus will always stack children vertically if not otherwise specified.

Boxes

Quickly design grid layouts.
